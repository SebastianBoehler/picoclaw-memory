# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Instead, report them privately to the maintainer with:

- a clear description of the issue
- affected versions or commits
- reproduction details
- impact assessment if known

You should receive an acknowledgement within a reasonable time, followed by triage and a remediation plan where appropriate.

## Scope

Security reports are especially useful for:

- auth or authorization flaws
- unsafe file handling
- injection risks
- dependency or container hardening issues
- data leakage between containers or projects
